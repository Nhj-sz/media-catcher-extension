# 抓媒酱

[![Test](https://github.com/Nhj-sz/media-catcher-extension/actions/workflows/test.yml/badge.svg)](https://github.com/Nhj-sz/media-catcher-extension/actions/workflows/test.yml)

这是一个自研的 Chrome/Edge 扩展，用来解决你提到的两个痛点：

1. 不想在 F12 里逐个点 open in new tab 再下载
2. 不知道多个 media 链接分别对应页面上的哪个视频

插件做法：

- 后台实时捕获当前标签页的媒体网络请求（media / 常见视频后缀 / 常见流媒体类型）
- 内容脚本扫描页面 video 元素与可疑媒体链接，提取标题、时长、分辨率等信息
- 在网页内注入悬浮面板，把“网络捕获 + 页面识别”合并展示，并标注来源
- 给每个 video 打上可点击编号角标（如 V1 / V2），支持精准定位
- 一键下载、复制链接、定位高亮（约 2 秒后自动消散）
- 支持图片元素定位（图片会以 I 编号显示）
- 仅播放中过滤
- 类型过滤开关（全部 / 仅视频 / 仅图片）
- 当前筛选结果批量下载
- 导出当前筛选链接（TXT / CSV / JSON）

## 功能清单

- 自动捕获媒体资源 URL
- 识别页面视频对应的标题（优先使用 video 周边语义文本）
- 网页内悬浮面板（无需先点浏览器扩展图标）
- 视频角标定位：可直接点击视频上的 V 编号定位目标
- 图片定位：可在面板定位图片元素（I 编号）
- 显示来源标签（页面元素 / 网络捕获）
- 显示基础信息（时长、大小、分辨率、MIME）
- 一键下载（直接走浏览器下载管理）
- 搜索筛选
- 类型过滤开关（全部 / 仅视频 / 仅图片）
- 仅播放中过滤
- 批量下载当前筛选结果
- 导出链接（TXT / CSV / JSON）
- 清空当前页缓存
- 支持捕获 iframe 内嵌页面中的视频 / 音频（子框架媒体也可识别与下载）
- 扫描页面 `<audio>` 元素并作为「音频」类型捕获与下载
- 录制下载：没有可下载直链时，可直接录制正在播放的视频 / 音频并保存为本地文件（视频导出 webm，音频导出 webm / ogg 等编码）
- 音频文件名智能识别：音频下载 / 录制默认使用 `.mp3` / `.m4a` / `.webm` 等音频扩展名，不再误标为 `.mp4`
- 流媒体识别：对 m3u8/mpd 等分片流标注「流媒体」并禁用直链下载，提示需借助专门合并工具
- 后台数据持久化：扩展 Service Worker 被浏览器回收后，已捕获的媒体记录不会丢失

## 获取源码

```bash
git clone https://github.com/Nhj-sz/media-catcher-extension.git
cd media-catcher-extension
```

本项目无需构建步骤：直接加载源码目录即可，修改后到 `chrome://extensions` 点“刷新”重新加载。

## 安装方式（开发者模式加载）

### Chrome

1. 打开 chrome://extensions/
2. 开启右上角“开发者模式”
3. 点击“加载已解压的扩展程序”
4. 选择扩展目录（即包含 `manifest.json` 的文件夹，例如克隆下来的 `media-catcher-extension` 目录）

### Edge

1. 打开 edge://extensions/
2. 开启左侧“开发人员模式”
3. 点击“加载解压缩的扩展”
4. 选择扩展目录（即包含 `manifest.json` 的文件夹）

## 使用步骤

1. 打开含视频的网页并开始播放（建议至少播放几秒）
2. 网页右下角会出现悬浮按钮“视频面板”（也可按 Alt+V）
3. 打开面板后点击“刷新”
4. 直接点击页面视频左上角的 V 编号（如 V1）或在面板点“定位”
5. 若目标是图片，可在面板定位 I 编号对应的图片元素
6. 确认高亮的是目标元素后点“下载”（定位高亮约 2 秒自动消散）
6. 可选：开启“仅播放中”快速过滤当前正在播放的视频
7. 可选：点击“类型: 全部”循环切换为“仅视频 / 仅图片”
8. 可选：点击“批量下载”下载当前筛选结果，或用“导出TXT/CSV/JSON”导出链接

## 注意事项

- 仅支持 http/https 直链下载。blob/data URL 无法直接下载（会自动禁用下载按钮）。
- 录制下载依赖浏览器 `MediaRecorder` 与媒体元素的 `captureStream` 能力：需先播放 1-2 秒再点「录制下载」，停止后自动保存本地文件。音频元素同样支持录制（导出 webm/ogg 等音频编码）。若浏览器或该媒体不支持捕获，按钮会给出对应提示。
- 对于 m3u8/mpd 这类分片流，浏览器下载的是清单文件本身，不是自动合并后的完整 MP4。面板已对这类条目标注「流媒体」并禁用直链下载按钮，避免误下清单；如需完整文件，请使用支持 HLS/DASH 的合并下载工具（如 yt-dlp / ffmpeg），可点条目「复制」拿到地址。
- 某些站点有 DRM 加密（如 EME/Widevine），链接可见但不可直接还原下载。
- 如果列表为空，通常是：
  - 页面尚未实际请求媒体
  - 站点做了较强的混淆或加密
  - 极少数极端跨域隔离场景下子框架媒体无法读取（多数 iframe 内视频 / 音频已可正常捕获与下载；仅「定位高亮」对子框架元素不可用，面板会提示直接下载）

## 文件结构

- manifest.json：扩展声明与权限
- background.js：网络捕获、数据聚合、下载调度
- content.js：页面媒体（视频 / 音频 / 图片）识别与上报、悬浮面板
- popup.html / popup.css / popup.js：弹窗 UI 与交互
- test/：Node 原生自动化测试（manifest / 后台 / 内容脚本 / 弹窗）

## 自动化测试

本扩展附带一套 Node 原生测试（无需浏览器），覆盖 manifest 校验、后台消息往返（DOM_MEDIA_FOUND → GET_MEDIA_FOR_TAB、DOWNLOAD_MEDIA 校验、持久化恢复）、内容脚本类型判定与 `buildPayload` 集成、弹窗纯函数等：

```bash
npm test
# 或
node --test
```

测试不依赖 Chrome，可在本地或 CI 快速回归。涉及真实页面交互（录制、定位高亮、批量下载）仍需手动在浏览器中验证。

## 权限与隐私

本扩展需要以下权限才能工作，请知悉其影响范围：

- `webRequest` + `host_permissions: <all_urls>`：用于实时监听当前标签页发出的媒体网络请求（仅读取响应头中的链接、类型、大小，不做任何拦截或修改）。这意味着扩展在启用期间**可以读取你访问的所有网站的媒体类网络请求**。
- `tabs` / `activeTab`：用于识别当前活动标签页并拉取其在读媒体。
- `downloads`：用于把捕获到的链接交给浏览器下载管理器。
- `storage`：用于把捕获到的媒体记录暂存在浏览器会话中（见下方说明），不写入任何账号或外部服务器。
- `scripting`：预留给后续可能的脚本注入能力。

数据存留说明：媒体记录仅保存在你本地浏览器中（内存 + `chrome.storage.session` 会话存储），**不上传任何服务器**。卸载扩展或关闭浏览器后，会话存储中的记录会自动清除。`storage.session` 的好处是扩展后台进程（Service Worker）被浏览器回收时，已捕获的记录不会随之丢失。
